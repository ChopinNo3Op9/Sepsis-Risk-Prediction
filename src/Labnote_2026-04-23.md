# Sepsis Experiment Notes

## Date
- 2026-04-23

## 1. Research Objective
- Continue the sepsis risk prediction study with a more complex deep sequence model than the baseline Transformer, and target higher prediction accuracy while keeping the same data windowing protocol.

## 2. Data Source
- PhysioNet Challenge 2019 (public ICU challenge training data)
- URL root: https://physionet.org/files/challenge-2019/1.0.0/training/
- Folders used: training_setA and training_setB
- Local data path: data/raw

## 3. Research Background and Theoretical Basis
- The baseline encoder-only Transformer captured temporal dependencies but did not achieve the best AUROC on this subset.
- A hybrid architecture can combine:
  - BiGRU recurrence for local temporal dynamics
  - Transformer self-attention for cross-time interactions
  - Attention pooling for adaptive summary over the 24-hour sequence
- Severe class imbalance suggests using focal loss and class weighting.

## 4. Experimental Procedures
- Script: src/experiment_hybrid_transformer.py
- Command:
  - "d:/Teach/NTU proj/Sepsis-Risk-Prediction/.venv/Scripts/python.exe" src/experiment_hybrid_transformer.py --data-dir data/raw --max-patients 220 --bootstrap-rounds 100
- Input features (base): HR, O2Sat, Temp, SBP, MAP, DBP, Resp, FiO2
- Feature engineering channels (per time step):
  - normalized values (z-score)
  - first-order temporal difference
  - missing-value indicator
- Final sequence input dimension: 24 channels (8 x 3)
- Labeling and windows:
  - history window: 24h
  - forecast horizon: 6h
  - label=1 if first sepsis onset in (t, t+6]
- Data split (sequence-level): 70% train / 15% val / 15% test
- Model:
  - input projection + LayerNorm + GELU
  - 2-layer BiGRU (bidirectional)
  - 2-layer Transformer encoder (d_model=96, nhead=4, FFN=256)
  - learnable temporal attention pooling
  - MLP head for binary classification
- Training:
  - loss: focal BCE with positive class weighting (gamma=2.0)
  - optimizer: AdamW (lr=8e-4, weight_decay=1e-3)
  - scheduler: cosine annealing
  - dropout: 0.2
  - early stopping patience: 4
  - gradient clipping: 1.0
- Threshold selection:
  - selected from validation set to maximize accuracy (tie-break by F1)
  - selected threshold: 0.65
- Test evaluation:
  - bootstrap with 100 rounds

## 6. Analysis and Conclusion
- The hybrid model achieved much higher raw accuracy by selecting a conservative threshold (0.65), effectively predicting nearly all windows as non-sepsis.
- This improved headline accuracy (0.9655 vs 0.7431 baseline Transformer) but collapsed clinical utility:
  - sensitivity and F1 dropped to 0
  - AUROC and AUPRC became worse than baseline
- Conclusion:
  - If objective is only overall accuracy on an imbalanced dataset, the new model + threshold strategy improved.
  - If objective is early sepsis detection quality, this configuration underperforms and needs threshold/optimization redesign.

## 7. Next Actions
- Re-select threshold using recall-constrained rule (for example sensitivity >= 0.7 on validation) instead of raw accuracy.
- Optimize by validation AUPRC or sensitivity-specificity tradeoff rather than accuracy.
- Run patient-level split to reduce leakage and obtain more realistic generalization estimates.
- Consider probability calibration (Platt scaling or isotonic) before threshold tuning.
