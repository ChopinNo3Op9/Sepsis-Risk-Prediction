# Sepsis Experiment Notes

## Date
- 2026-04-16

## 1. Research Objective
- Research Objective: evaluate a single-model Transformer encoder for first sepsis onset in next 6 hours using 24-hour history and document bootstrap performance against prior baselines (Logistic Regression, GBT, LSTM).

## 2. Data Source
- PhysioNet Challenge 2019 (public ICU challenge training data)
- URL root: https://physionet.org/files/challenge-2019/1.0.0/training/
- Folders used: training_setA and training_setB
- Local data path: data/raw

## 3. Research Background and Theoretical Basis
- Sepsis is an acute life-threatening response to infection; early detection is critical.
- Physiological vitals from PhysioNet provide temporal signals with missingness patterns.
- Transformer encoders use multi-head self-attention to model long-range dependencies across the 24-hour window, unlike the LSTM which processes tokens sequentially and is biased toward recent time steps.
- The proposal (TFTSepsis) motivates attention-based architectures for ICU time series; this experiment implements a lightweight encoder-only variant as a stepping stone.

## 4. Experimental Procedures
- Preprocess: read p*.psv files, causal forward fill, train-set median imputation for remaining NaNs, z-score normalization with train statistics.
- Features: HR, O2Sat, Temp, SBP, MAP, DBP, Resp, FiO2
- Label: 1 if first sepsis onset occurs within (t, t+6] after 24h history window.
- Split: 70/15/15 sequence-based (with 1867/400/401 samples from 2668 total windows).
- Model: 1-layer Transformer Encoder (d_model=64, nhead=4, dim_feedforward=128), sinusoidal positional encoding, LayerNorm, last-step output → 2-layer MLP classifier (64 → GELU → 1).
- Training: BCEWithLogitsLoss with positive-class weighting, Adam optimizer, batch_size=128, epochs=8, learning_rate=0.001, gradient clipping (max_norm=1.0), early stopping patience=8.
- Model selection: best epoch by validation AUROC.
- Bootstrap metrics on test: 100 rounds.

### Architecture decisions explored during development
| Variant | Val AUROC | Test AUROC | Note |
|---|---|---|---|
| CLS token at position 0 + causal mask | 0.500 | 0.500 | CLS cannot attend to any input tokens |
| CLS at end + causal mask | 0.797 | 0.429 | Severe overfitting |
| Mean pooling (bidirectional) | 0.802 | 0.460 | Signal diluted across 24 time steps |
| Last-step pooling (bidirectional) | 0.762 | 0.616 | Best generalization |
| Heavy regularization (dropout=0.3, LR=0.0002) | 0.831 | 0.529 | Better val but worse test |

### Key debugging finding
- The initial implementation was missing the **causal forward fill** step in `make_samples_for_patient`, which all other experiments (LogReg, GBT, LSTM) include. Without forward fill, many NaN values were replaced by the global median, destroying within-patient temporal continuity and costing approximately 0.10–0.15 test AUROC.

## 6. Analysis and Conclusion
- Conclusion: the Transformer achieves test AUROC 0.616, ranking second behind the LSTM (0.690) and ahead of Logistic Regression (0.577) and GBT (0.496) on this subset.
- The Transformer shows better calibration (Brier 0.166, ECE 0.336) than the LSTM (Brier 0.251, ECE 0.466), but weaker discrimination.
- Unlike the LSTM which achieved perfect sensitivity with poor specificity, the Transformer has moderate sensitivity (0.28) and specificity (0.76), suggesting it is less aggressively biased toward predicting positive.
- The main challenge was overfitting: with only 220 patients (2668 windows, ~56 positives), the attention mechanism memorizes training patterns rapidly. Training loss dropped from 1.18 to 0.04 within 8 epochs.
- Architecture choice mattered significantly: placing the CLS token before the sequence with a causal mask prevented the model from learning (AUROC=0.50); switching to last-step pooling (analogous to LSTM final hidden state) yielded the best generalization.
- Limitations: only 220 patients, sequence-based split instead of patient-level split, fixed threshold=0.5, no calibration tuning, no hyperparameter search.
- Next steps: scale to full patient set, implement patient-level split, add dropout/weight-decay sweep, threshold optimization on validation set, and explore the full TFT architecture with static covariates and missingness embeddings as proposed.
