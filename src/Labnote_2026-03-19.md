# Sepsis Experiment Notes (Reproducible Log)

## Date
- 2026-03-19

## 1. Experiment Title/Research Objective
- Title: Reproducible Early Sepsis Risk Forecasting Benchmark on PhysioNet 2019 ICU Data (Logistic Regression Only)
- Research Objective: evaluate single-model Logistic Regression for first sepsis onset in next 6 hours using 24-hour history and document bootstrap performance.

## 2. Data Source
- PhysioNet Challenge 2019 (public ICU challenge training data)
- URL root: https://physionet.org/files/challenge-2019/1.0.0/training/
- Folders used: training_setA and training_setB
- Local data path: data/raw

## 3. Research Background/Theoretical Basis
- Sepsis is acute life-threatening response to infection; early detection is critical.
- Physiological vitals from PhysioNet provide temporal signals with missingness patterns.
- Logistic regression on flattened 24h time blocks is a robust baseline before more complex architectures.

## 4. Experimental Procedures
- Preprocess: read p*.psv files, causal forward fill, median imputation (train median) for remaining NaNs, z-score standardization.
- Features: HR, O2Sat, Temp, SBP, MAP, DBP, Resp, FiO2
- Label: 1 if first sepsis onset occurs within (t, t+6] after 24h history window.
- Split: 70/15/15 sequence-based (with 1867/400/401 samples from 2668 total windows).
- Model: LogisticRegression(max_iter=1200, class_weight='balanced', solver='lbfgs').
- Bootstrap metrics on test: 100 rounds.

## 5. Results and Observation Records

## 6. Analysis and Conclusion
- Conclusion: baseline logistic regression yields limited sensitivity although specificity is high; class imbalance and low prevalence make clinical-positive detection hard.
- Limitations: only 220 patients, simplified split strategy, no patient-level index tracking in windows, no calibration threshold tuning.
- Next steps: scale to more patients, add XGBoost/LSTM comparison, perform patient-level splits, and threshold optimization.
